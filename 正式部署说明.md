# 正式部署说明

## 为什么刚才那个固定链接还不够

`loca.lt` 这种方式只是把你电脑上的网页临时暴露到外网。

它的问题是：

- 你电脑关机，别人就打不开
- 你不启动服务，别人也打不开
- 虽然地址能固定，但本质还是“借你电脑提供网页”

你现在真正需要的是：

- 电脑关机后也能访问
- 地址长期稳定
- 你更新照片墙、案例库、联系方式登记后，线上也同步保留

## 这次给你准备的正式方案

我已经把项目改成更适合云端长期托管的结构，并添加了：

- `render.yaml`
- `requirements.txt`
- `runtime.txt`

推荐正式部署到 **Render**，原因是：

- 适合这种带图片上传和本地持久存储的小型站点
- 可以挂持久磁盘，案例图、工作照片墙、登记名单不会因为重启就丢
- 部署后给你一个真正长期可访问的网址

## 部署完成后会得到什么

你会得到：

1. 一个长期稳定的网址
2. 游客主页长期可访问
3. 登记联系方式长期可用
4. 案例库、照片墙、名单数据持续保留

## 你后面怎么更新

部署完成后，你平时的更新逻辑会变成两种方式里的其中一种：

### 方式 A：继续本地改，再重新部署

适合你偶尔更新的时候。

### 方式 B：我再帮你做一个在线后台

这样你以后在网页后台里直接上传案例图、替换照片墙，不需要重新部署。

## 下一步我建议

下一步我直接帮你做：

1. 初始化正式部署目录
2. 安装 Render CLI 或用网页方式部署
3. 把现在这套网站正式上线
4. 给你最终长期网址

## 需要注意

Render 的持久磁盘通常不是免费能力，所以真正“长期稳定”这一步，大概率需要你有一个可用的 Render 账号，并接受它的付费小站方案。
